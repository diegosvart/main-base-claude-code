# main-base-claude-code — Onboarding

You are helping a developer understand and use the `main-base-claude-code` template.
Read these instructions and guide them through the onboarding.

---

## What is this template

`main-base-claude-code` is a Claude Code project template that works in two ways:

1. **For new projects**: an inception agent runs a structured discovery session
   before any code is written, producing architecture and planning artifacts,
   then bootstraps a project with 6 pre-configured layers.

2. **For existing projects**: an analyze mode scans the repo, detects gaps vs
   the template standard, and proposes incremental adoption.

---

## Quick orientation

Ask the developer: **"What do you want to do?"**

Then guide them to the right command:

### Starting a new project from scratch
```bash
# Step 1: install the inception skill globally (one-time setup)
mkdir -p ~/.claude/skills/inception
curl -sL https://raw.githubusercontent.com/diegosvart/main-base-claude-code/main/skill/SKILL.md \
  -o ~/.claude/skills/inception/SKILL.md

# Step 2: from any empty directory
/inception "describe your project idea here"
```

### Recovering interrupted work
```bash
# From the project directory
/inception --resume
```

### Analyzing an existing project
```bash
# From the project directory or with a path
/inception --analyze .
```

### Using the template directly (without inception)
```bash
gh repo create my-project --template diegosvart/main-base-claude-code --private
cd my-project
# Edit CLAUDE.md with your project details
# Configure .mcp.json if needed
```

---

## The 6 layers

Show the developer what's pre-configured:

1. **CLAUDE.md** — project memory (generated by inception or filled manually)
2. **Skills** (`.claude/skills/`) — playbooks loaded on demand
3. **Commands** (`.claude/commands/`) — `/commit`, `/new-feature`, `/create-pr`, `/review`, `/close-phase`
4. **Hooks** (`.claude/settings.json`) — auto-format on write, session log, notifications
5. **MCP** (`.mcp.json`) — empty by default, inception adds servers per stack
6. **Agents** (`.claude/agents/`) — TL/Orchestrator + Architecture + Backend + DBA + Frontend

---

## Key workflow

```
/inception "idea" → discovery (4 phases) → bootstrap → /new-feature → code → /review → /create-pr → merge → /close-phase
```

---

## Recommended first step

If the developer has a project idea:
→ Run `/inception "their idea"` — the agent will guide the rest.

If they're exploring the template:
→ Show them `machine-readable/index.yaml` for a compact overview of everything available.

If they have an existing project:
→ Run `/inception --analyze .` to see what gaps exist.
